# エージェント設定・管理ツール 要件定義（MVP）

## 1. ドキュメントの目的

本ドキュメントは、**Go + Bubble Tea を用いた TUI ベースのエージェント設定・管理ツール**について、現時点で確定している事項に基づき **MVP 要件** を定義するものである。

---

## 2. プロダクト概要

本ツールは、**Claude Code** および **Codex** 向けの設定資産を、**TUI（Terminal User Interface）上で横断的に閲覧し、テンプレートベースで管理するための個人向けツール**である。

本ツール自体には **LLM を搭載しない**。そのため、本ツールは推論や会話を行うエージェントではなく、設定資産の **閲覧・切替・差分確認・テンプレート選択/作成支援** を担う。

---

## 3. MVP の前提

### 3.1 技術方針

* 実装言語は **Go** とする。
* TUI フレームワークとして **Bubble Tea** を採用する。
* 本ツールは **TUI 中心の CLI アプリケーション**として提供する。

### 3.2 利用対象

* 本ツールの MVP は **個人利用** を前提とする。
* 初期対応 OS は **Linux のみ** とする。

### 3.3 対応対象

* 対応対象は以下の 2 つとする。

  * **Claude Code**
  * **Codex**

### 3.4 プロダクト制約

* 本ツール自体には **LLM を搭載しない**。
* Marketplace 連携は **MVP に含めない**。
* TUI 内の直接編集機能は **MVP に含めない**。
* 非対話コマンド（例: `scan`, `validate`）は **MVP に含めない**。

---

## 4. 管理対象

MVP における管理対象資産は以下とする。

* **Skills**
* **Agents**
* **MCP**
* **`AGENTS.md`**
* **`CLAUDE.md`**

settings 系および profile 系は **MVP の管理対象に含めない**。

---

## 5. スコープとプロジェクト管理要件

### 5.1 スコープ種別

* 本ツールは少なくとも以下 2 つのスコープを扱う。

  * **Global**
  * **Project**

### 5.2 Project の管理方式

* Project は、現在の作業ディレクトリ自動認識ではなく、**本アプリに登録して管理する方式**とする。
* ユーザーは複数の Project を本アプリ内で保持し、切り替えて扱えること。
* Project 管理の体験は、**VS Code のワークスペース一覧に近い感覚**を想定する。

### 5.3 ナビゲーション方式

* `Global / Project` をタブで切り替える方式は採用しない。
* TUI の **サイドバー上部に Global を表示し、その下に登録済み Project 一覧を表示する**構成とする。
* ユーザーはサイドバー上で **Global または任意の Project を選択**できること。

### 5.4 選択対象の扱い

* ユーザーが **Global** を選択している場合も、**Project** を選択している場合も、選択対象に対応する設定内容を **メインエリアで閲覧できる**こと。
* 選択対象に応じて、メインエリアの表示内容が切り替わること。

### 5.5 優先順位・差分

* MVP で **Global / Project 間の差分および優先順位を可視化**すること。
* 差分表示の粒度は、MVP では **ファイル単位** とする。
* ユーザーが、どのスコープの設定が有効であるかを確認できること。

---

## 6. UI 要件

### 6.1 UI 方式

* 本ツールは、単発コマンド中心ではなく、**近年の対話型 CLI アプリに近い TUI** を提供する。

### 6.2 Provider 切替

* **Claude Code / Codex** の切替を TUI 上で **タブ UI** として提供する。

### 6.3 画面構成

* 画面構成は、**一覧と詳細を同時表示する構成**とする。
* 少なくとも以下の 2 領域を持つこと。

  * **サイドバー**: Global および登録済み Project 一覧を表示する
  * **メインエリア**: 選択対象の設定内容を表示する
* メインエリアでは、管理対象資産を **グルーピングして表示**できること。

### 6.4 閲覧表示

* 選択中の Global または Project に対して、管理対象資産を TUI 上で一覧表示できること。
* 一覧の対象には少なくとも Skills、Agents、MCP、`AGENTS.md`、`CLAUDE.md` を含むこと。

### 6.5 詳細表示

* 選択した資産の詳細をメインエリア上で確認できること。
* 詳細表示は **常時表示** とすること。
* 詳細表示には、少なくとも以下を含むこと。

  * 資産種別
  * 対象 Provider
  * 対象 Scope
  * 実体の場所
  * 有効状態
  * 差分または優先関係

## 7. 機能要件

### 7.1 閲覧機能

* Claude Code / Codex 向けの管理対象資産を、単一の TUI 上で閲覧できること。
* Provider 切替およびサイドバー上の対象切替に応じて、表示内容が変わること。

### 7.2 差分・優先順位の可視化

* Global / Project に同種の資産がある場合、その差分や優先関係を確認できること。
* どちらが有効かをユーザーが把握できること。
* 差分表示は MVP ではファイル単位で行うこと。

### 7.3 テンプレート作成

* MVP では、資産の新規作成を **テンプレートベース** で行うこと。
* テンプレート作成対象は、MVP では **ディレクトリ構造のみ** とする。
* テンプレートは **名前付きで保存し、再利用可能** であること。
* ユーザー自身がテンプレートを新規定義して保存する機能は **MVP に含めない**。
* テンプレート作成フローは、ユーザーの選択操作を通じて構成を決める方式とする。
* テンプレート作成 UI では、少なくとも以下を満たすこと。

  * テンプレート作成開始操作があること
  * サイドバーに **Global および各 Project の一覧** が表示されること
  * メインエリアに **Skills / Agents / MCP / `AGENTS.md` / `CLAUDE.md`** などがグルーピングされて表示されること
  * 各項目を **チェックボックスで選択**できること
  * 対象 Provider として **Codex / Claude Code** をチェックボックスで選択できること
* テンプレート適用結果は、選択対象の **Global または Project に即時書き出す**こと。

### 7.4 非対象機能

MVP では以下を対象外とする。

* Marketplace 検索・導入
* TUI 内での直接編集
* 非対話モードでのコマンド実行
* チーム配布・共有・標準化機能
* settings / profile 系の管理

---

## 8. 非機能要件

### 8.1 実装制約

* 実装は Go を用いること。
* TUI は Bubble Tea を用いること。

### 8.2 利用環境

* 初期対応環境は Linux のみとする。

### 8.3 アプリケーション特性

* 本ツールは LLM を内蔵しないこと。
* 振る舞いは可能な限り明示的かつ決定的であること。

---

## 9. MVP スコープ外

以下は MVP に含めない。

* settings / profile 系の管理
* Marketplace / registry 連携
* TUI 内編集
* 非対話コマンド
* チーム向け配布
* Windows / macOS 対応

---

## 10. Project 登録管理要件

* Project の追加および削除は **TUI 内で行える**こと。
* 登録済み Project はサイドバー一覧に反映されること。

---

## 11. 未確定事項

現時点で、MVP 要件として未確定の事項はない。
